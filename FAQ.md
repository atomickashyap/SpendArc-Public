# ❓ Frequently Asked Questions

---

### Is SpendArc open source?

**No.** SpendArc is a proprietary application. The source code is not publicly available.

---

### Can I build or run the app from SpendArc‑Public?

**No.** SpendArc‑Public contains only public-facing documentation, feature descriptions, and screenshots. It does not contain any application code, build files, or configuration needed to compile or run SpendArc.

---

### Is the source code available anywhere?

The source code is **not publicly available**. SpendArc‑Public is a product overview — not a code repository.

---

### How can I get the app?

SpendArc is currently in development for public release. Check [CONTACT.md](CONTACT.md) for updates, early access, or waitlist information.

---

### What platforms does SpendArc support?

SpendArc is currently available for **Android**. Other platforms may be explored in the future — see [ROADMAP.md](ROADMAP.md).

---

### Is my financial data sent to the cloud?

**No.** All your financial data stays on your device. SpendArc does not sync to any cloud service. See [PRIVACY.md](PRIVACY.md) for full details.

---

### Does SpendArc require SMS permission?

**No.** SMS permission is optional. If granted, SpendArc can automatically read bank transaction messages to record expenses. The app works fully without SMS access — you can enter everything manually.

---

### Does SpendArc store my SMS messages?

**No.** SpendArc never stores raw SMS messages. It only extracts transaction details (amount, date, merchant) and discards the original message content.

---

### How does the AI assistant work?

The AI assistant lets you ask questions about your spending in natural language. Your questions are sent to a third-party AI service to generate responses. No conversation data is stored externally after the response is generated.

---

### Is SpendArc free?

**Yes.** SpendArc is completely free to use. As the app grows and evolves, we may explore optional ways to sustain the project — such as non-intrusive ads or premium features — depending on the user base and ongoing development costs. Any changes will be communicated transparently, and core functionality will always remain accessible.
